## Context

P05-01 created the `ChipRegistry` and `ChipDefinition` types. P05-18 created `evaluateChip()` for recursive hierarchy evaluation. But there is no way to **place** a registered chip on the 3D canvas as a visual component that can be wired.

Currently, the only placeable components are the 7 primitive gate types (`GateType = 'NAND' | 'AND' | ...`) and I/O nodes. This ticket adds a new visual component — a "composite chip box" — that represents any `ChipDefinition` from the registry.

### Design decisions:

1. **Visual**: A labeled rectangular box (wider than a primitive gate) with the chip name on top, input pins on the left edge, output pins on the right edge
2. **Store representation**: A new entity type alongside `GateInstance`, or extend `GateInstance` with a `chipName` field and dynamic pin generation
3. **Wiring**: Pin positions are dynamic based on the chip's `inputs`/`outputs` arrays. The existing wire system connects to pins by `entityId + pinId`.

### Recommended approach:

Extend the existing `GateInstance` type with an optional `chipName: string` field. When `chipName` is set, the gate renders as a composite chip box instead of a primitive gate visual. Pin IDs come from the `ChipDefinition`'s pin names.

---

## Goal

After this ticket:
1. `placeCompositeChip(chipName, position)` store action creates a gate-like instance with pins from the registry
2. `CompositeChip3D` renders a labeled box with input pins on left, output pins on right
3. Pins are wirable using the existing wire system
4. Evaluation: when `simulationTick` encounters a composite chip, it calls `evaluateChip()` (P05-18) and sets output pin values
5. Composite chips appear in the gate selector (filtered from `ChipRegistry.list()`)

### `src/gates/components/CompositeChip3D.tsx`

```tsx
import { Text } from '@react-three/drei'
import type { GateInstance, Position } from '@/store/types'

interface Props {
  gate: GateInstance
  chipName: string
  inputPins: Array<{ id: string; name: string; value: number }>
  outputPins: Array<{ id: string; name: string; value: number }>
  isSelected: boolean
  onPointerDown?: (e: any) => void
}

export function CompositeChip3D({ gate, chipName, inputPins, outputPins, isSelected, onPointerDown }: Props) {
  const pinCount = Math.max(inputPins.length, outputPins.length)
  const boxHeight = Math.max(2, pinCount * 1.2)
  const boxWidth = 3.0
  const boxDepth = 1.5

  return (
    <group
      position={[gate.position.x, gate.position.y, gate.position.z]}
      rotation={[gate.rotation.x, gate.rotation.y, gate.rotation.z]}
      onPointerDown={onPointerDown}
    >
      {/* Main box */}
      <mesh>
        <boxGeometry args={[boxWidth, boxDepth, boxHeight]} />
        <meshStandardMaterial
          color={isSelected ? '#4a9eff' : '#2a5a8a'}
          transparent
          opacity={0.9}
        />
      </mesh>

      {/* Chip name on top */}
      <Text
        position={[0, boxDepth / 2 + 0.01, 0]}
        rotation={[-Math.PI / 2, 0, 0]}
        fontSize={0.4}
        color="#ffffff"
        anchorX="center"
        anchorY="middle"
      >
        {chipName}
      </Text>

      {/* Input pins (left edge) */}
      {inputPins.map((pin, i) => {
        const z = (i - (inputPins.length - 1) / 2) * 1.2
        return (
          <group key={pin.id} position={[-boxWidth / 2, 0, z]}>
            <mesh>
              <sphereGeometry args={[0.15]} />
              <meshStandardMaterial color={pin.value !== 0 ? '#00ff88' : '#666'} />
            </mesh>
            <Text position={[-0.4, 0.3, 0]} fontSize={0.2} color="#aaa" anchorX="right">
              {pin.name}
            </Text>
          </group>
        )
      })}

      {/* Output pins (right edge) */}
      {outputPins.map((pin, i) => {
        const z = (i - (outputPins.length - 1) / 2) * 1.2
        return (
          <group key={pin.id} position={[boxWidth / 2, 0, z]}>
            <mesh>
              <sphereGeometry args={[0.15]} />
              <meshStandardMaterial color={pin.value !== 0 ? '#00ff88' : '#666'} />
            </mesh>
            <Text position={[0.4, 0.3, 0]} fontSize={0.2} color="#aaa" anchorX="left">
              {pin.name}
            </Text>
          </group>
        )
      })}
    </group>
  )
}
```

### `src/store/actions/compositeChipActions/compositeChipActions.ts`

```typescript
import type { GateInstance, Position } from '@/store/types'
import { getStudentChipRegistry } from '@/core/chips/appRegistry'

export interface CompositeChipActions {
  placeCompositeChip: (chipName: string, position: Position) => GateInstance | null
}

export const createCompositeChipActions = (set: any, get: any): CompositeChipActions => ({
  placeCompositeChip: (chipName: string, position: Position) => {
    // Look up ChipDefinition from registry
    // Create a GateInstance with chipName field and dynamic pins
    // Pins: one per input/output from ChipDefinition
    let result: GateInstance | null = null

    set((state: any) => {
      const registry = getStudentChipRegistry()
      const chipDef = registry.get(chipName)
      if (!chipDef) return

      const id = `chip-${chipName}-${Date.now()}`
      const inputs = chipDef.inputs.map((pin: any, i: number) => ({
        id: `${id}-in-${pin.name}`,
        name: pin.name,
        type: 'input' as const,
        value: 0,
      }))
      const outputs = chipDef.outputs.map((pin: any, i: number) => ({
        id: `${id}-out-${pin.name}`,
        name: pin.name,
        type: 'output' as const,
        value: 0,
      }))

      const gate: GateInstance = {
        id,
        type: 'COMPOSITE' as any, // or extend GateType union
        chipName,
        position,
        rotation: { x: 0, y: 0, z: 0 },
        inputs,
        outputs,
      }

      state.gates.push(gate)
      result = gate
    }, false, 'placeCompositeChip')

    return result
  },
})
```

## Pitfalls

- **`GateType` expansion**: Adding `'COMPOSITE'` to the `GateType` union may break exhaustive switch statements in existing code (`gateLogic.ts`, 3D components). Either add a case or use `chipName !== undefined` as the discriminator and leave `type` as `'NAND'` (less clean but safer).
- **Pin ID generation**: Composite chip pins need stable IDs for the wire system. Use `${gateId}-in-${pinName}` format. Pin IDs must not collide with primitive gate pin IDs.
- **Pin positions for wiring**: The wire system uses `getPinWorldPosition(gateId, pinId)`. Composite chips must implement this — calculate pin positions based on the chip's position + pin index offset.
- **Registry access in store**: The `ChipRegistry` is not in Zustand state (it contains closures). Prefer the same **`getStudentChipRegistry()`** module as P05-21 — avoid fake `state._chipRegistry` on Immer drafts.
- **Pin `value` type**: After P05-02, gate pins use **`number`**; composite chip pins should match (0 = off for visualization).
- **HDL / registry names**: `placeCompositeChip('Not')` looks up **`ChipDefinition.name`** — must match HDL chip names and `getChipFiles` keys (case-sensitive).
- **Box sizing**: Dynamic height based on pin count. Use `Math.max(inputCount, outputCount) * 1.2` for the z-dimension.
- **React Three Fiber**: `<Text>` comes from `@react-three/drei`. Already a dependency.
