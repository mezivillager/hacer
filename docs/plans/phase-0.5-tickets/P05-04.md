## Context

HACER has no HDL support. The directory `src/core/hdl/` does not exist. Students need to write Hardware Description Language code to define chips textually. The parser is the first step — it turns HDL source text into a structured AST that the compiler (P05-16, a later ticket) will use.

The **nand2tetris HACK HDL** dialect is small and well-specified. All 16 Project 1 `.hdl` files are available in the sibling workspace at `web-ide/projects/src/project_01/*.ts`, each exporting `{ hdl, tst, cmp }` string constants. For this ticket, use the HDL strings directly as test fixtures.

## HDL Grammar (Project 1 scope)

```
ChipDecl    = 'CHIP' Name '{' InDecl OutDecl PartsDecl '}'
InDecl      = 'IN' PinList ';'
OutDecl     = 'OUT' PinList ';'
PinList     = Pin (',' Pin)*
Pin         = Name ('[' Number ']')?
PartsDecl   = 'PARTS:' (PartInst ';')* | 'BUILTIN' Name ';'
PartInst    = Name '(' ConnList ')'
ConnList    = Conn (',' Conn)*
Conn        = Name '=' (Name SubBus? | 'true' | 'false')
SubBus      = '[' Number ('..' Number)? ']'
Name        = [A-Za-z_][A-Za-z0-9_]*
Number      = [0-9]+
Comment     = '//' ... EOL | '/*' ... '*/' | '/**' ... '*/'
```

## Files to Create

### `src/core/hdl/types.ts`

```typescript
export interface HDLPin {
  name: string
  width: number  // default 1; e.g., `a[16]` → width=16
}

export interface HDLConnection {
  internal: string   // part pin name (e.g., "a", "in", "out")
  external: string   // wire/chip-pin name (e.g., "a", "nandOut", "true", "false")
  start?: number     // sub-bus start: a[3] → start=3; a[0..7] → start=0
  end?: number       // sub-bus end:   a[3] → end=3;   a[0..7] → end=7
}

export interface HDLPart {
  name: string             // chip name (e.g., "Nand", "Not")
  connections: HDLConnection[]
}

export interface HDLChip {
  name: string
  inputs: HDLPin[]
  outputs: HDLPin[]
  parts: HDLPart[]
  builtin?: string  // if `BUILTIN Nand;` instead of PARTS
  clocked?: string[] // future: for sequential logic (Phase 0.6)
}

export interface HDLParseError {
  line: number
  column: number
  message: string
}

export type HDLParseResult =
  | { success: true; chip: HDLChip }
  | { success: false; errors: HDLParseError[] }
```

#### Token types needed:

```typescript
type TokenType =
  | 'CHIP' | 'IN' | 'OUT' | 'PARTS' | 'BUILTIN'
  | 'TRUE' | 'FALSE'
  | 'IDENT' | 'NUMBER'
  | 'LBRACE' | 'RBRACE' | 'LPAREN' | 'RPAREN'
  | 'LBRACKET' | 'RBRACKET'
  | 'SEMICOLON' | 'COMMA' | 'EQUALS' | 'COLON' | 'DOTDOT'
  | 'EOF'
```

#### Tokenizer rules:
- Skip whitespace and comments (`//` to EOL, `/*` to `*/`, `/**` to `*/`)
- Keywords: `CHIP`, `IN`, `OUT`, `PARTS`, `BUILTIN`, `true`, `false`
- Identifiers: `[A-Za-z_][A-Za-z0-9_]*`
- Numbers: `[0-9]+`
- Symbols: `{`, `}`, `(`, `)`, `[`, `]`, `;`, `,`, `=`, `:`, `..`
- Track line/column for error reporting

#### Parser structure (recursive descent):
```
parseChip()     → expect 'CHIP' IDENT '{' parseIN() parseOUT() parsePARTS() '}'
parseIN()       → expect 'IN' parsePinList() ';'
parseOUT()      → expect 'OUT' parsePinList() ';'
parsePinList()  → parsePin() (',' parsePin())*
parsePin()      → IDENT ('[' NUMBER ']')?
parsePARTS()    → expect 'PARTS' ':' (parsePart() ';')* | 'BUILTIN' IDENT ';'
parsePart()     → IDENT '(' parseConnList() ')'
parseConnList() → parseConn() (',' parseConn())*
parseConn()     → IDENT '=' (IDENT parseSubBus()? | 'true' | 'false')
parseSubBus()   → '[' NUMBER ('..' NUMBER)? ']'
```

```typescript
import { describe, it, expect } from 'vitest'
import { parseHDL } from './parser'

// Inline compatibility-fixture HDL for self-contained tests
const NAND_HDL = `CHIP Nand {
    IN a, b;
    OUT out;
    PARTS:
    BUILTIN Nand;
}`

const NOT_HDL = `CHIP Not {
    IN in;
    OUT out;
    PARTS:
    //// Replace this comment with your code.
}`

const MUX4WAY16_HDL = `CHIP Mux4Way16 {
    IN a[16], b[16], c[16], d[16], sel[2];
    OUT out[16];
    PARTS:
    //// Replace this comment with your code.
}`

describe('HDL Parser', () => {
  describe('basic parsing', () => {
    it('parses Nand BUILTIN declaration', () => {
      const result = parseHDL(NAND_HDL)
      expect(result.success).toBe(true)
      if (!result.success) return
      expect(result.chip.name).toBe('Nand')
      expect(result.chip.inputs).toEqual([
        { name: 'a', width: 1 },
        { name: 'b', width: 1 },
      ])
      expect(result.chip.outputs).toEqual([{ name: 'out', width: 1 }])
      expect(result.chip.builtin).toBe('Nand')
    })

    it('parses Not stub (empty PARTS)', () => {
      const result = parseHDL(NOT_HDL)
      expect(result.success).toBe(true)
      if (!result.success) return
      expect(result.chip.name).toBe('Not')
      expect(result.chip.inputs).toEqual([{ name: 'in', width: 1 }])
      expect(result.chip.outputs).toEqual([{ name: 'out', width: 1 }])
      expect(result.chip.parts).toEqual([])
    })

    it('parses Mux4Way16 with bus widths and multi-bit sel', () => {
      const result = parseHDL(MUX4WAY16_HDL)
      expect(result.success).toBe(true)
      if (!result.success) return
      expect(result.chip.name).toBe('Mux4Way16')
      expect(result.chip.inputs).toHaveLength(5)
      expect(result.chip.inputs[0]).toEqual({ name: 'a', width: 16 })

      expect(result.chip.outputs).toEqual([{ name: 'out', width: 16 }])
    })

  describe('parts and connections', () => {
    it('parses a chip with parts and connections', () => {
      const hdl = `CHIP Not {
        IN in;
        OUT out;
        PARTS:
        Nand(a=in, b=in, out=out);

      const result = parseHDL(hdl)
      expect(result.success).toBe(true)
      if (!result.success) return
      expect(result.chip.parts).toHaveLength(1)
      expect(result.chip.parts[0].name).toBe('Nand')
      expect(result.chip.parts[0].connections).toEqual([
        { internal: 'a', external: 'in' },
        { internal: 'b', external: 'in' },
        { internal: 'out', external: 'out' },

    })

    it('parses sub-bus connections', () => {

      expect(result.success).toBe(true)
      if (!result.success) return
      expect(result.chip.parts[0].connections[0]).toEqual({

        IN in[16];

        PARTS:

      const result = parseHDL(hdl)
      expect(result.success).toBe(true)

        internal: 'in', external: 'in', start: 3, end: 3,

      const hdl = `CHIP Test {
        IN in;
        OUT out;
        PARTS:
        Nand(a=in, b=true, out=out);

      const result = parseHDL(hdl)
      expect(result.success).toBe(true)

      expect(result.chip.parts[0].connections[1]).toEqual({
        internal: 'b', external: 'true',

    })

      const hdl = `// This is a comment

        IN in; // input

        PARTS:
        // implementation

      const result = parseHDL(hdl)
      expect(result.success).toBe(true)

      const result = parseHDL(hdl)
      expect(result.success).toBe(true)
```

## Compatibility HDL Examples

**Nand (BUILTIN):**
```hdl
CHIP Nand { IN a, b; OUT out; PARTS: BUILTIN Nand; }
```

**Not (stub — empty PARTS):**
```hdl
CHIP Not { IN in; OUT out; PARTS: //// Replace this comment with your code. }
```

**Mux4Way16 (bus widths):**
```hdl
CHIP Mux4Way16 {
    IN a[16], b[16], c[16], d[16], sel[2];
    OUT out[16];
    PARTS: //// Replace this comment with your code.
}
```

**A completed Not (what students write):**
```hdl
CHIP Not { IN in; OUT out; PARTS: Nand(a=in, b=in, out=out); }
```

**A completed And (uses internal wire):**
```hdl
CHIP And { IN a, b; OUT out; PARTS:
    Nand(a=a, b=b, out=nandOut);
    Not(in=nandOut, out=out);
}
```

## Pitfalls

- **`BUILTIN Nand;`** is a special case — no part list, just a single keyword. The `Nand.hdl` file uses this. The parser should set `chip.builtin = 'Nand'` and `chip.parts = []`.
- **`//// Replace this comment`** — four slashes. This is just a line comment. Make sure the tokenizer handles `////` as a comment.
- **`PARTS:` followed by nothing** — valid (empty parts list for stubs). Don't require at least one part.
- **Pin names with numbers** — e.g., `nandOut` is a valid identifier. Don't reject digits in the middle.
- **Multiple parts separated by `;`** — each part instantiation ends with `;`.
- **The `:` after `PARTS`** — it's `PARTS:`, not `PARTS {`. The tokenizer should handle `PARTS` as a keyword and `:` as a separate token.
- **No `src/core/` exists** — create the full directory tree.
- **Vite alias** — `@/core/hdl/...` works via the `@/` → `./src/*` alias.
