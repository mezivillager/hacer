## Context

P05-05 created the TST parser (`parseTST` → `TSTScript`), P05-06 created the CMP parser (`parseCMP` → `CmpFile`), and P05-03 delivered correct single-pass evaluation. This ticket builds the engine that connects them: run a `.tst` script against a chip, record outputs, and compare against `.cmp` data.

### Test script execution flow:

```
1. `load Chip.hdl` → resolve chip from registry
2. `output-list a b out` → configure output columns
3. `set a 0, set b 0` → set chip inputs
4. `eval` → evaluate chip (single pass)
5. `output` → record current output values as one row
6. Repeat steps 3–5 for all test cases
7. Compare recorded output rows against .cmp data
```

### TSTCommand types (from P05-05):

- `load`: resolve chip by name
- `output-file`: set output file name (informational)
- `compare-to`: load .cmp data
- `output-list`: configure output columns with format specifiers
- `set`: set a chip input pin to a value
- `eval`: evaluate the chip once
- `output`: record output row
- `repeat` / `while`: control flow (Project 1 uses these for some chips)

## Files to Create

### `src/core/testing/nand2tetris/engine.ts`

**Align with P05-05 / P05-06:** `parseTST` returns `TSTParseResult` (unwrap `success` → `script`). CMP types use `CmpColumn[]` and `CmpRow.values: number[]`; prefer reusing **`compareCmpRow(actualValues, expectedRow, columns)`** from `cmpParser.ts` instead of hand-rolling index lookup on a stale shape.

```typescript
import type { TSTScript, TSTCommand } from './types'
import type { CmpFile } from './cmpParser' // or './types' if CmpFile is re-exported there
import { compareCmpRow } from './cmpParser'

export interface ChipEvaluator {
  setInput(name: string, value: number): void
  evaluate(): void
  /** Return `undefined` if this chip has no output pin with that name (so engine can fall back to `getInput`). */
  getOutput(name: string): number | undefined
  getInput(name: string): number
}

export interface ChipResolver {
  resolve(chipName: string): ChipEvaluator | null
}

export interface OutputRow {
  values: Record<string, number>
}

export interface TestFailure {
  row: number
  column: string
  expected: string
  actual: string
}

export interface TestResult {
  passed: boolean
  totalSteps: number
  passedSteps: number
  outputRows: OutputRow[]
  firstFailure: TestFailure | null
  error: string | null // runtime error message
}

export function runTest(
  script: TSTScript,
  resolver: ChipResolver,
  options?: {
    /** If omitted, engine still runs but skips CMP checks unless `compare-to` is implemented below. */
    cmpData?: CmpFile
    /** Optional: resolve `compare-to Foo.cmp` from bundled curriculum (P05-07). */
    loadCmpFile?: (fileName: string) => CmpFile | null
  }
): TestResult {
  const outputRows: OutputRow[] = []
  let chip: ChipEvaluator | null = null
  let outputColumns: string[] = []
  let cmpRowIndex = 0
  let cmpData: CmpFile | undefined = options?.cmpData

  for (const cmd of script.commands) {
    switch (cmd.type) {
      case 'load':
        chip = resolver.resolve(cmd.chipName)
        if (!chip) {
          return {
            passed: false, totalSteps: 0, passedSteps: 0,
            outputRows, firstFailure: null,
            error: `Chip "${cmd.chipName}" not found`,
          }
        }
        break

      case 'compare-to':
        if (options?.loadCmpFile) {
          cmpData = options.loadCmpFile(cmd.fileName) ?? undefined
          cmpRowIndex = 0
        }
        break

      case 'output-list':
        outputColumns = cmd.columns.map(c => c.name)
        break

      case 'set':
        if (!chip) break
        chip.setInput(cmd.pin, cmd.value)
        break

      case 'eval':
        if (!chip) break
        chip.evaluate()
        break

      case 'output': {
        if (!chip) break
        const row: Record<string, number> = {}
        for (const col of outputColumns) {
          const outVal = chip.getOutput(col)
          const val = outVal !== undefined ? outVal : chip.getInput(col)
          row[col] = val
        }
        outputRows.push({ values: row })

        if (cmpData && cmpRowIndex < cmpData.rows.length) {
          // Order must match `cmpData.columns` (see P05-06), not only `output-list` iteration order.
          const actualRow = cmpData.columns.map((c) => row[c.name] ?? 0)
          const mismatch = compareCmpRow(actualRow, cmpData.rows[cmpRowIndex], cmpData.columns)
          if (mismatch) {
            return {
              passed: false,
              totalSteps: outputRows.length,
              passedSteps: cmpRowIndex,
              outputRows,
              firstFailure: {
                row: cmpRowIndex, // authoritative row for this `output` step (see P05-06 pitfall if helper omits it)
                column: mismatch.column,
                expected: String(mismatch.expected),
                actual: String(mismatch.actual),
              },
              error: null,
            }
          }
          cmpRowIndex++
        }
        break
      }

      case 'repeat':
        // Execute cmd.body `count` times; each iteration must run the SAME command runner as the outer loop (extract `runCommand(cmd)`).
        // Pseudocode only — do not ship a stub that returns `{ ok: true }` without executing body.
        break

      case 'while':
        // Project 1 may not use `while`; if P05-05 parses it, implement or return structured `error: 'while not supported'`.
        break

      // output-file: informational for UI / export; optional to store on TestResult
    }
  }

  if (cmpData && cmpRowIndex !== cmpData.rows.length) {
    return {
      passed: false,
      totalSteps: outputRows.length,
      passedSteps: cmpRowIndex,
      outputRows,
      firstFailure: null,
      error: `Output row count ${cmpRowIndex} does not match .cmp row count ${cmpData.rows.length}`,
    }
  }

  return {
    passed: true,
    totalSteps: outputRows.length,
    passedSteps: outputRows.length,
    outputRows,
    firstFailure: null,
    error: null,
  }
}
```

**API note:** Prefer a single internal function `dispatchCommands(commands, ctx)` mutating shared `outputRows` / `cmpRowIndex` so `repeat` / nested blocks do not duplicate logic.

function createEvaluatorFromRegistry(registry: ChipRegistry): ChipResolver {
  return {
    resolve(chipName: string): ChipEvaluator | null {
      const chipDef = registry.get(chipName)
      if (!chipDef || !isBuiltinChip(chipDef)) return null

      const inputs: Record<string, number> = {}
      const outputs: Record<string, number> = {}

      for (const pin of chipDef.inputs) inputs[pin.name] = 0
      for (const pin of chipDef.outputs) outputs[pin.name] = 0

      return {
        setInput(name, value) { inputs[name] = value },
        evaluate() {
          const result = chipDef.implementation.evaluate({ ...inputs })
          Object.assign(outputs, result)
        },
        getOutput(name) { return outputs[name] },
        getInput(name) { return inputs[name] ?? 0 },
      }
    },
  }
}

## Pitfalls

- **TST parser shape**: The exact `TSTCommand` discriminated union types depend on P05-05. Read the TST parser types file before implementing. The `parseTST` return value shape is critical.
- **CMP parser shape (P05-06):** `CmpFile.columns` is `CmpColumn[]` (name + index), `CmpRow.values` is `number[]`. Use **`parseCmp`** (not `parseCMP`) and **`compareCmpRow`** for each `output` step.
- **`%S` runtime semantics**: `output-list` columns using `%S` (for example `time%S1.3.1`) require runtime support, not parser-only support. The engine must provide a string time source with tick/tock phase fidelity (for example `0+`, `1`, `1+`), apply string-style output formatting, and compare `%S` cells without numeric coercion.
- **Input vs output columns**: Some `.tst` files include input pins in `output-list`. Use **`getOutput(col)` only when defined** (`number | undefined`), then fall back to `getInput(col)` — do not use `??` on `0` from `getOutput`.
- **`repeat` / `while`**: Must share one implementation with the top-level loop. **Do not** leave a stub `executeCommands` that always returns success.
- **Row-count parity**: After the script, if `cmpData` was used, `cmpRowIndex` must equal `cmpData.rows.length` or return `passed: false` with `error` message.
- **`compare-to` AST field**: Use the property name from P05-05 (`fileName` vs `path` vs `filename`) — the sketch uses `cmd.fileName`.
- **Value formatting**: `.cmp` files use specific formats (`%B`, `%D`). The comparison must parse `.cmp` cell values into numbers for comparison, not string comparison. P05-06's parser should already return numeric values.
- **P05-07 dependency**: If the curriculum data bundle isn't merged yet, inline test fixtures for Not, And, and Mux. Add the comprehensive 16-chip test once the data is available.
- **Performance**: Each test has 2–16 rows (Project 1). The engine should handle the entire suite in <1s.
