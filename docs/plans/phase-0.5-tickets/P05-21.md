## Context

P05-04 created the HDL parser (`parseHDL`), P05-16 created the compiler (`compileHDL`). Together they form the `HDL source → AST → ChipDefinition` pipeline. This ticket wraps that pipeline in a code editor panel so students can type HDL and see it compile (or see errors) in real time.

### Current App layout:

```tsx
<Layout>
  <Sidebar />
  <CanvasArea />
  <DemoOverlay />
</Layout>
```

The HDL editor will be a new panel. Two possible layouts:
- **Tabbed**: CanvasArea has tabs for "3D Canvas" and "HDL Editor" (simpler, recommended for Layer 3)
- **Side-by-side**: CanvasArea splits horizontally showing 3D + editor (future enhancement)

### Integration with chip workflow browser (P05-19):

When a student selects a chip from the workflow browser and clicks "Start", the HDL editor should pre-fill with that chip's starter HDL template from a fixture/provider source.

## Goal

After this ticket:
1. `HDLEditor` component renders a code editor with HDL syntax highlighting
2. Auto-compile on edit (debounced ~500ms): parse → compile → register in ChipRegistry or show errors

4. On successful compile: chip registered in `ChipRegistry` (accessible to P05-17 test engine and P05-18 hierarchy eval)

## Files to Create

### `src/components/ui/HDLEditor.tsx`

```tsx
import { circuitActions } from '@/store/circuitStore'
import { useRef, useState } from 'react'
import { parseHDL } from '@/core/hdl/parser'
import { compileHDL } from '@/core/hdl/compiler'
// Registry: module-level singleton (e.g. `src/core/chips/appRegistry.ts`) — NOT Zustand state.
import { getStudentChipRegistry, registerOrReplaceChip } from '@/core/chips/appRegistry'

export interface CompileError {
  line: number
  column: number
  message: string
}

export interface CompileResult {
  success: boolean
  chipName?: string
  errors: CompileError[]
}

export function HDLEditor() {
  const [source, setSource] = useState('')
  const [errors, setErrors] = useState<CompileError[]>([])
  const [lastCompileResult, setLastCompileResult] = useState<CompileResult | null>(null)
  const debounceRef = useRef<ReturnType<typeof setTimeout> | null>(null)

  const handleChange = (newSource: string) => {
    setSource(newSource)
    if (debounceRef.current) clearTimeout(debounceRef.current)
    debounceRef.current = setTimeout(() => {
      compileSource(newSource)
    }, 500)
  }

  const compileSource = (src: string) => {
    if (!src.trim()) {
      setErrors([])
      setLastCompileResult(null)
      return
    }

    // Align with P05-04: `HDLParseResult` is `{ success: true; chip } | { success: false; errors }`
    const parseResult = parseHDL(src)
    if (!parseResult.success) {
      const errs = parseResult.errors.map(e => ({
        line: e.line,
        column: e.column,
        message: e.message,
      }))
      setErrors(errs)
      setLastCompileResult({ success: false, errors: errs })
      circuitActions.addStatus?.('error', errs[0]?.message ?? 'HDL parse failed')
      return
    }

    const registry = getStudentChipRegistry() // must already include Nand + builtins per P05-15/P05-01
    const compileResult = compileHDL(parseResult.chip, registry)
    if (compileResult.type !== 'success') {
      const errs = compileResult.errors.map(e => ({
        line: e.line ?? 0,
        column: e.column ?? 0,
        message: e.message,
      }))
      setErrors(errs)
      setLastCompileResult({ success: false, errors: errs })
      circuitActions.addStatus?.('error', errs[0]?.message ?? 'HDL compile failed')
      return
    }

    registerOrReplaceChip(registry, compileResult.chip) // see Pitfalls: re-registration

    setErrors([])
    setLastCompileResult({ success: true, chipName: compileResult.chip.name, errors: [] })
    circuitActions.addStatus?.('success', `${compileResult.chip.name} compiled`)
  }

  // Render: textarea as minimal editor, or CodeMirror if dependency added
  return (
    <div data-testid="hdl-editor" style={{ display: 'flex', flexDirection: 'column', height: '100%' }}>
      <div style={{ padding: '4px 8px', fontSize: 12, borderBottom: '1px solid #333' }}>
        <span>HDL Editor</span>
        {lastCompileResult?.success && (
          <span style={{ color: '#52c41a', marginLeft: 8 }}>
            ✓ {lastCompileResult.chipName} compiled
          </span>
        )}
      </div>
      <textarea
        data-testid="hdl-editor-textarea"
        value={source}
        onChange={e => handleChange(e.target.value)}
        style={{
          flex: 1, fontFamily: 'monospace', fontSize: 13,
          background: '#1a1a2e', color: '#e0e0e0', border: 'none',
          padding: 8, resize: 'none', outline: 'none',
        }}
        spellCheck={false}
      />
      {errors.length > 0 && (
        <div data-testid="hdl-editor-errors" style={{ padding: '4px 8px', background: '#2a0a0a', fontSize: 11 }}>
          {errors.map((e, i) => (
            <div key={i} style={{ color: '#ff4d4f' }}>
              Line {e.line}: {e.message}
            </div>
          ))}
        </div>
      )}
    </div>
  )
}
```

**CodeMirror upgrade path:** Start with a plain `<textarea>` for correctness. Once the compile loop works, swap in CodeMirror 6 (`@codemirror/view`, `@codemirror/state`) with a custom HDL language mode. Install via `pnpm add @codemirror/view @codemirror/state @codemirror/language`. The language mode highlights: `CHIP`, `IN`, `OUT`, `PARTS`, `BUILTIN` as keywords; identifiers, numbers, brackets, `//` and `/* */` comments.

## Pitfalls

- **CodeMirror dependency**: Start without it (plain textarea). CodeMirror 6 requires `@codemirror/view`, `@codemirror/state`, and optionally `@codemirror/language` for the custom HDL mode. Do not add CodeMirror until the compile loop works with the textarea.
- **Registry singleton vs store**: `ChipRegistry` contains closures (`evaluate` functions) and is **not JSON-serializable**. Do not put it in Zustand state. Use a module-level singleton or a React context.
- **Debounce on every keystroke**: 500ms debounce prevents compile on every character. Use `setTimeout` + `clearTimeout` pattern, not `setInterval`.
- **Compile error line numbers**: `parseHDL` errors include `line` and `column`. The compile step (P05-16) may not — if compile errors lack positions, map them using the AST node that caused the error.
- **Re-registration**: Compiling the same chip name twice should update the registry, not throw "already registered." Either add `registry.unregister(name)` + `register`, or `replace(name, def)` on `ChipRegistry`.
- **`addStatus` / optional chaining**: The sketch uses `circuitActions.addStatus?.(...)` until P05-09 exposes the method on the facade — remove `?.` once wired.
- **Starter template → editor**: Prefer lifting `selectedChipName` + `hdlTemplate` via store or a small React context shared with P05-19 so `HDLEditor` can `setSource(...)` without circular imports.
- **React Compiler**: No `useMemo` or `useCallback`. The debounce ref is fine as `useRef`.
