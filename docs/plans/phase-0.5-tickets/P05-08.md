## Scope Refresh (2026-03-29)

This ticket was written earlier and had stale assumptions. Scope is refreshed to current codebase and decisions:

1. **Include a user-facing rename control now** in the Sidebar (not only console/store APIs).
2. **Validation feedback must be visible** from the UI path using Ant Design feedback patterns.
3. **Uniqueness is case-insensitive** within each node type bucket (`input` vs `output`).
4. **Name format is constrained to HDL-safe identifiers**: `[A-Za-z_][A-Za-z0-9_]*`.
5. **Boundary remains strict**: no chip definition panel, auto-IO generation, or interface-level validation here (P05-20).

## Context

Current code facts:

1. `InputNode` and `OutputNode` already include `name: string` in `src/store/types.ts`.
2. `NodeActions` currently has no rename methods.
3. `InputNode3D.tsx` and `OutputNode3D.tsx` receive `name` but discard it (`name: _name`) and only render value text.
4. Sidebar currently supports node placement and deletion only; no node rename UI exists.
5. `circuitActions` is exposed to E2E through `window.__CIRCUIT_ACTIONS__`; new rename methods must be added to action typing parity.

---

## Goal

After this ticket:

1. `circuitActions.renameInputNode(nodeId, newName)` and `renameOutputNode(...)` are available.
2. Rename enforces: trim, non-empty, HDL identifier format, case-insensitive uniqueness in same node type.
3. Selected nodes can be renamed from Sidebar UI (not only via console).
4. Invalid rename attempts show user-visible feedback (UI path).
5. Input and output node names are rendered as visible labels in the 3D scene, alongside signal value display.

## Pitfalls / Constraints

- Keep store actions deterministic and safe: invalid rename is a no-op in state mutation.
- Keep UI feedback in UI layer; do not add user-facing side effects directly in store action functions.
- Case-insensitive duplicate check applies only within same node type bucket (`input` with `input`, `output` with `output`).
- Do not change node placement auto-name strategy in this ticket.
- Do not expand into chip interface validation or generation flows from P05-20.
