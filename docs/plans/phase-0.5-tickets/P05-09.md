## Context

HACER has no error/status reporting system. When things go wrong (invalid wiring, test failures, parse errors), the student gets no feedback. The gap analysis notes: "No status bar with success/error messages, no validation of incomplete circuits, no warning for floating inputs, no error messages from test execution."

The web-ide shows a status bar with severity-colored messages like "Simulation successful" or "Simulation error at line N."

### Current app layout (`src/App.tsx`):

```tsx
function AppContent() {
  useKeyboardShortcuts()
  return (
    <Layout className="app-layout">
      <Sidebar />
      <CanvasArea />
      <DemoOverlay />
    </Layout>
  )
}
```

Ant Design's `Layout` component is already in use. The StatusBar will be added as a footer within the Layout.

---

## Goal

After this ticket:
1. A `StatusBar` component renders at the bottom of the app
2. Store actions `addStatus(severity, text)` and `clearStatus(id)` manage messages
3. Messages are severity-colored: info (blue), warning (yellow), error (red)
4. Messages can be dismissed by clicking
5. Later tickets (test results, validation, HDL errors) will post messages via these actions

### `src/store/actions/statusActions/statusActions.ts`

```typescript
import type { StatusActions, CircuitStore } from '../../types'

type SetState = (fn: (state: CircuitStore) => void, replace?: false, actionName?: string) => void

let statusCounter = 0

export const createStatusActions = (set: SetState): StatusActions => ({
  addStatus: (severity, text) => {
    const id = `status-${++statusCounter}`
    const message = { id, severity, text, timestamp: Date.now() }
    set((state) => {
      state.statusMessages.push(message)
    }, false, 'addStatus')
    return message
  },

  clearStatus: (id) => {
    set((state) => {
      state.statusMessages = state.statusMessages.filter(m => m.id !== id)
    }, false, 'clearStatus')
  },

  clearAllStatus: () => {
    set((state) => {
      state.statusMessages = []
    }, false, 'clearAllStatus')
  },
})
```

### `src/components/ui/StatusBar.tsx`

```tsx
import { useCircuitStore } from '@/store/circuitStore'
import { circuitActions } from '@/store/circuitStore'

const SEVERITY_COLORS = {
  info: '#1890ff',
  warning: '#faad14',
  error: '#ff4d4f',
} as const

export function StatusBar() {
  const messages = useCircuitStore(state => state.statusMessages)
  const latest = messages[messages.length - 1]

  if (!latest) return null

  return (
    <div
      style={{
        position: 'fixed',
        bottom: 0,
        left: 0,
        right: 0,
        padding: '4px 12px',
        backgroundColor: SEVERITY_COLORS[latest.severity],
        color: '#fff',
        fontSize: 13,
        display: 'flex',
        justifyContent: 'space-between',
        alignItems: 'center',
        zIndex: 1000,
        cursor: 'pointer',
      }}
      onClick={() => circuitActions.clearStatus(latest.id)}
      data-testid="status-bar"
    >
      <span data-testid="status-text">{latest.text}</span>
      <span style={{ opacity: 0.7, marginLeft: 8 }}>click to dismiss</span>
    </div>
  )
}
```

    junctionPlacementMode: null,

    junctionPreviewWireId: null,
    statusMessages: [],
  })

beforeEach(() => {
  resetStore()
})

describe('statusActions', () => {
  it('addStatus adds a message to statusMessages', () => {
    getState().addStatus('info', 'Hello')
    expect(getState().statusMessages).toHaveLength(1)
    expect(getState().statusMessages[0].severity).toBe('info')
    expect(getState().statusMessages[0].text).toBe('Hello')
  })

  it('addStatus returns the created message with id', () => {
    const msg = getState().addStatus('error', 'Broken')
    expect(msg.id).toBeTruthy()
    expect(msg.severity).toBe('error')
  })

  it('addStatus appends (does not replace)', () => {
    getState().addStatus('info', 'First')
    getState().addStatus('warning', 'Second')
    expect(getState().statusMessages).toHaveLength(2)
  })

  it('clearStatus removes a specific message by id', () => {
    const msg = getState().addStatus('info', 'Remove me')
    getState().addStatus('error', 'Keep me')
    getState().clearStatus(msg.id)
    expect(getState().statusMessages).toHaveLength(1)
    expect(getState().statusMessages[0].text).toBe('Keep me')
  })

  it('clearAllStatus removes all messages', () => {
    getState().addStatus('info', 'One')
    getState().addStatus('warning', 'Two')
    getState().clearAllStatus()
    expect(getState().statusMessages).toHaveLength(0)
  })

  it('clearStatus is no-op for unknown id', () => {
    getState().addStatus('info', 'Keep')
    getState().clearStatus('nonexistent')
    expect(getState().statusMessages).toHaveLength(1)
  })
})

## Pitfalls

- **Initial state**: Add `statusMessages: []` to the initial state in `circuitStore.ts`. If you forget, the store will have `undefined` for `statusMessages`.
- **Immer compatibility**: `state.statusMessages.push(message)` works with Immer drafts. Do not spread to create new arrays — mutate in place.
- **`CircuitStore` type**: The combined type must include `StatusActions`. Follow the pattern of existing action interfaces (e.g., `GateActions`, `SimulationActions`).
- **`circuitActions` facade**: Add the three new actions to the `circuitActions` object in `circuitStore.ts`, following the same pattern as existing actions.
- **React 19 + React Compiler**: No `useMemo` or `useCallback`. The `useCircuitStore(state => state.statusMessages)` selector is sufficient.
- **Test rendering**: `StatusBar` is a plain React component (not R3F). Standard `@testing-library/react` `render()` works.
- **E2E globals**: `circuitStore.ts` assigns `window.__CIRCUIT_ACTIONS__ = circuitActions` — once `addStatus` / `clearStatus` / `clearAllStatus` are on that object, Playwright can call them. Update **`e2e/types/globals.ts`** (`CircuitActionsAPI`, `CircuitStoreSnapshot`) so `statusMessages` and the new actions are typed.
- **Layout**: `StatusBar` uses `position: fixed` at the bottom — it does not have to be `Layout.Footer`; sibling of `Sidebar`/`CanvasArea` inside `Layout` is fine.
