## Files to Create

### `docs/guides/project1-workflow.md`

Student-facing guide covering:

1. **Getting started**: Open HACER → select "Project 1: Boolean Logic" in sidebar
2. **Understanding chip specs**: Click a chip → see IN/OUT pins, description
3. **HDL workflow**:
   - Click "Start" → HDL editor loads stub
   - Write implementation in PARTS section
   - See auto-compile result (green ✓ or red error)
   - Click "Run Test" → see output table
   - Green = done, move to next chip
4. **3D workflow**:
   - Click "Start" → I/O nodes auto-generated on canvas
   - Place gates from selector
   - Wire gates to I/O nodes
   - Click "Validate" → check I/O matches spec
   - Click "Run Test" → see results
5. **Recommended build order**: Not → And → Or → Xor → Mux → DMux → Not16 → And16 → Or16 → Mux16 → Or8Way → Mux4Way16 → Mux8Way16 → DMux4Way → DMux8Way
6. **Tips**:
   - Use builtin toggle to experiment before implementing
   - Save your work frequently (circuits auto-save, but named saves are safer)
   - Each chip builds on previously-built chips — complete them in order
   - For 16-bit chips: use bus splitters/joiners to manage wide connections
