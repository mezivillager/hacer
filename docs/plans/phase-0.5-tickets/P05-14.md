## Context

HACER stores all circuit state in Zustand (in-memory). Refreshing the browser destroys everything. Project 1 has 15 chips, each taking 15–60 minutes — users need multi-session persistence.

After P05-03 (topological sort), the simulation state is correct and worth saving. The serialization format must capture:
- All gates with types, positions, rotations, pin values
- All wires with endpoints, segments, signalId
- All I/O nodes with names, positions, values, widths
- All junctions with signalId and wireIds

### Current state shape (from `circuitStore.ts`):

Gates, wires, inputNodes, outputNodes, junctions — these are the serializable circuit data. UI state (selectedGateId, placementMode, etc.) is NOT serialized.

---

## Goal

After this ticket:
1. `serializeCircuit(state)` → `SerializedCircuit` JSON object with version field
2. `deserializeCircuit(data)` → partial `CircuitState` ready to load into the store
3. `saveCircuit(name)` → persists to `localStorage` under a namespaced key
4. `loadCircuit(name)` → restores circuit state from localStorage
5. `listSavedCircuits()` → returns array of saved circuit names with timestamps
6. `deleteSavedCircuit(name)` → removes from localStorage
7. Auto-save on every meaningful state change (debounced)
8. Export JSON file download + Import JSON file upload
9. `CircuitLibrary` UI component: list, save, load, delete, export, import

### `src/core/serialization/serialize.ts`

```typescript
import type { CircuitState } from '@/store/types'
import type { SerializedCircuit } from './types'

export function serializeCircuit(state: CircuitState, name: string): SerializedCircuit {
  return {
    version: 1,
    name,
    savedAt: new Date().toISOString(),
    gates: state.gates.map(g => ({
      id: g.id,
      type: g.type,
      position: { ...g.position },
      rotation: { ...g.rotation },
    })),
    wires: state.wires.map(w => ({
      id: w.id,
      signalId: w.signalId,
      from: { type: w.from.type, entityId: w.from.entityId, pinId: w.from.pinId },
      to: { type: w.to.type, entityId: w.to.entityId, pinId: w.to.pinId },
      segments: w.segments.map(s => ({ start: { ...s.start }, end: { ...s.end }, axis: s.axis })),
      crossesWireIds: [...w.crossesWireIds],
    })),
    inputNodes: state.inputNodes.map(n => ({
      id: n.id, name: n.name, position: { ...n.position },
      rotation: { ...n.rotation }, value: Number(n.value), width: n.width,
    })),
    outputNodes: state.outputNodes.map(n => ({
      id: n.id, name: n.name, position: { ...n.position },
      rotation: { ...n.rotation }, value: Number(n.value), width: n.width,
    })),
    junctions: state.junctions.map(j => ({
      id: j.id, position: { ...j.position },
      signalId: j.signalId, wireIds: [...j.wireIds],
    })),
  }
}
```

## Pitfalls

- **Immer drafts are not serializable**: `serializeCircuit` must receive a plain state snapshot (`useCircuitStore.getState()`), NOT an Immer draft. Never call it from inside `set()`.
- **localStorage size**: Each circuit is ~10–50KB JSON. `localStorage` has a 5–10MB limit per origin. Sufficient for dozens of circuits.
- **Gate pin reconstruction**: Pins are not serialized — they must be reconstructed during deserialization. If P05-12 (bus components) or P05-02 (number values) change pin shapes, keep the reconstruction logic in sync.
- **Auto-save debounce**: Use `subscribeWithSelector` to watch `gates`, `wires`, `inputNodes`, `outputNodes`, `junctions`. Debounce 2s. Save to a `__autosave__` key.
- **`CircuitStore` type**: Add `PersistenceActions` to the `CircuitStore` union. `listSavedCircuits` is a read-only helper — it reads localStorage directly without mutating store state.
- **Export/Import**: Export calls `serializeCircuit` and triggers a `Blob` download. Import reads a `File`, parses JSON, validates version, and calls `loadCircuit` logic.
