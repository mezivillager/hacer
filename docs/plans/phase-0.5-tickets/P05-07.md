## Context

Nand2tetris Project 1 requires 48 curriculum files: 16 `.hdl` stubs, 16 `.tst` test scripts, and 16 `.cmp` expected outputs (15 student-built chips + the Nand primitive).

These files exist in the **sibling workspace** at `web-ide/projects/src/project_01/`. Each chip is a TypeScript module exporting `{ hdl, tst, cmp }` as string constants. The directory `src/data/` does not exist in HACER yet.

The nand2tetris course materials are licensed under CC BY-NC-SA 3.0.

## Files to Create

### `src/data/nand2tetris/project01/` — one `.ts` file per chip

Copy the string constants from the web-ide source. Each file exports `{ hdl, tst, cmp }`:

```typescript
// src/data/nand2tetris/project01/nand.ts
export const hdl = `CHIP Nand { ... }`
export const tst = `output-list a b out; ...`
export const cmp = `| a | b |out| ...`
```

Use lowercase filenames: `nand.ts`, `not.ts`, `and.ts`, `or.ts`, `xor.ts`, `mux.ts`, `dmux.ts`, `not16.ts`, `and16.ts`, `or16.ts`, `mux16.ts`, `mux4way16.ts`, `mux8way16.ts`, `dmux4way.ts`, `dmux8way.ts`, `or8way.ts`.

### `src/data/nand2tetris/project01/index.ts`

```typescript
import * as Nand from './nand'
import * as Not from './not'
import * as And from './and'
import * as Or from './or'
import * as Xor from './xor'
import * as Mux from './mux'
import * as DMux from './dmux'
import * as Not16 from './not16'
import * as And16 from './and16'
import * as Or16 from './or16'
import * as Mux16 from './mux16'
import * as Mux4Way16 from './mux4way16'
import * as Mux8Way16 from './mux8way16'
import * as DMux4Way from './dmux4way'
import * as DMux8Way from './dmux8way'
import * as Or8Way from './or8way'

export interface ChipFiles {
  hdl: string
  tst: string
  cmp: string
}

const chipData: Record<string, ChipFiles> = {
  Nand, Not, And, Or, Xor, Mux, DMux,
  Not16, And16, Or16, Mux16,
  Mux4Way16, Mux8Way16,
  DMux4Way, DMux8Way, Or8Way,
}

export const CHIP_ORDER: string[] = [
  'Not', 'And', 'Or', 'Xor', 'Mux', 'DMux',
  'Not16', 'And16', 'Or16', 'Mux16',
  'Or8Way', 'Mux4Way16', 'Mux8Way16', 'DMux4Way', 'DMux8Way',
]

export const ALL_CHIPS: string[] = ['Nand', ...CHIP_ORDER]

export function getChipFiles(chipName: string): ChipFiles | undefined {
  return chipData[chipName]
}

export function getChipNames(): string[] {
  return ALL_CHIPS
}
```

## Test Plan

### `src/data/nand2tetris/project01/index.test.ts`

```typescript
import { describe, it, expect } from 'vitest'
import { getChipFiles, getChipNames, CHIP_ORDER, ALL_CHIPS } from './index'

describe('Project 1 curriculum data', () => {
  it('ALL_CHIPS has 16 entries', () => {
    expect(ALL_CHIPS).toHaveLength(16)
  })

  it('CHIP_ORDER has 15 entries (excludes Nand)', () => {
    expect(CHIP_ORDER).toHaveLength(15)
    expect(CHIP_ORDER).not.toContain('Nand')
  })

  it('CHIP_ORDER starts with Not and ends with DMux8Way', () => {
    expect(CHIP_ORDER[0]).toBe('Not')
    expect(CHIP_ORDER[CHIP_ORDER.length - 1]).toBe('DMux8Way')
  })

  for (const name of ['Nand', 'Not', 'And', 'Or', 'Xor', 'Mux', 'DMux',
    'Not16', 'And16', 'Or16', 'Mux16', 'Mux4Way16', 'Mux8Way16',
    'DMux4Way', 'DMux8Way', 'Or8Way']) {
    describe(`${name}`, () => {
      it('getChipFiles returns non-empty hdl/tst/cmp', () => {
        const files = getChipFiles(name)
        expect(files).toBeDefined()
        expect(files!.hdl.length).toBeGreaterThan(0)
        expect(files!.tst.length).toBeGreaterThan(0)
        expect(files!.cmp.length).toBeGreaterThan(0)
      })

      it('hdl contains CHIP declaration', () => {
        const files = getChipFiles(name)!
        expect(files.hdl).toContain(`CHIP ${name}`)
      })

      it('cmp starts with pipe delimiter', () => {
        const files = getChipFiles(name)!
        expect(files.cmp.trim().startsWith('|')).toBe(true)
      })
    })
  }

  it('getChipFiles returns undefined for unknown chip', () => {
    expect(getChipFiles('Unknown')).toBeUndefined()
  })

  it('getChipNames returns all 16 names', () => {
    expect(getChipNames()).toHaveLength(16)
  })
})
```

## Pitfalls

- **String escaping**: The HDL/TST/CMP strings contain backticks and special chars. Use template literals (backtick strings) for multi-line content, same as the web-ide source.
- **Nand has no `.hdl` stub** in the traditional sense — its HDL uses `BUILTIN Nand;`. But the web-ide still exports an `hdl` string for it, so include it.
- **CHIP_ORDER excludes Nand** — students don't build Nand; it's the provided primitive.
- **`src/data/` does not exist** — create the full directory tree.
- **Don't modify the source strings** — copy them exactly from the web-ide. The parsers (P05-04, P05-05, P05-06) must handle the original format.
- **Import the web-ide data as reference only** — the HACER bundle should be self-contained (no runtime dependency on the web-ide workspace).
