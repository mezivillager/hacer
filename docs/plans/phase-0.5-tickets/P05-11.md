## Context

After P05-02, all values are `number` but the simulation still treats everything as single-bit (0 or 1). Multi-bit buses (e.g. `in[16]`) need bitmask operations to read/write sub-buses and the simulation must propagate integer values wider than 1 bit through wires.

The `Wire` type in `src/store/types.ts` currently has no `width` field. Nodes have `width: number` but the wire system ignores it.

### Current wire structure (`src/store/types.ts`):

```typescript
export interface Wire {
  id: string
  signalId?: string
  from: WireEndpoint
  to: WireEndpoint
  segments: WireSegment[]
  crossesWireIds: string[]
}
```

## Goal

After this ticket:
1. `readSubBus(value, startBit, numBits)` extracts bits from a bitmask
2. `writeSubBus(target, value, startBit, numBits)` writes bits into a bitmask
3. `Wire` type has an optional `width` field (default 1)
4. The simulation (topological eval or `simulationTick`) propagates multi-bit values through wires
5. A 16-bit InputNode wired to a 16-bit OutputNode propagates the full value in a single eval

## Files to Create

### `src/simulation/busOps.ts`

```typescript
/**
 * Create a bitmask with `numBits` ones starting at bit 0.
 * maskForWidth(8) => 0xFF, maskForWidth(16) => 0xFFFF
 */
export function maskForWidth(numBits: number): number {
  if (numBits <= 0) return 0
  if (numBits >= 32) return 0xFFFFFFFF >>> 0
  return ((1 << numBits) - 1) >>> 0
}

/**
 * Read `numBits` bits starting at `startBit` from `value`.
 * readSubBus(0xFF00, 8, 8) => 0xFF
 */
export function readSubBus(value: number, startBit: number, numBits: number): number {
  return ((value >>> startBit) & maskForWidth(numBits)) >>> 0
}

/**
 * Write `numBits` bits of `subValue` into `target` starting at `startBit`.
 * writeSubBus(0, 0xFF, 8, 8) => 0xFF00
 */
export function writeSubBus(target: number, subValue: number, startBit: number, numBits: number): number {
  const mask = maskForWidth(numBits) << startBit
  return ((target & ~mask) | ((subValue << startBit) & mask)) >>> 0
}

/**
 * Clamp a value to the given bit width.
 * clampToWidth(0x1FFFF, 16) => 0xFFFF
 */
export function clampToWidth(value: number, width: number): number {
  return (value & maskForWidth(width)) >>> 0
}
```

### `src/simulation/busOps.test.ts`

```typescript
import { describe, it, expect } from 'vitest'
import { maskForWidth, readSubBus, writeSubBus, clampToWidth } from './busOps'

describe('maskForWidth', () => {
  it('width 1 → 0x1', () => expect(maskForWidth(1)).toBe(1))
  it('width 8 → 0xFF', () => expect(maskForWidth(8)).toBe(0xFF))
  it('width 16 → 0xFFFF', () => expect(maskForWidth(16)).toBe(0xFFFF))
  it('width 0 → 0', () => expect(maskForWidth(0)).toBe(0))
})

describe('readSubBus', () => {
  it('reads lower byte from 0xFF00', () => {
    expect(readSubBus(0xFF00, 0, 8)).toBe(0)
  })
  it('reads upper byte from 0xFF00', () => {
    expect(readSubBus(0xFF00, 8, 8)).toBe(0xFF)
  })
  it('reads single bit', () => {
    expect(readSubBus(0b1010, 1, 1)).toBe(1)
    expect(readSubBus(0b1010, 0, 1)).toBe(0)
  })
  it('reads full 16 bits', () => {
    expect(readSubBus(0x1234, 0, 16)).toBe(0x1234)
  })
})

describe('writeSubBus', () => {
  it('writes upper byte', () => {
    expect(writeSubBus(0, 0xFF, 8, 8)).toBe(0xFF00)
  })
  it('writes lower byte preserving upper', () => {
    expect(writeSubBus(0xFF00, 0xAB, 0, 8)).toBe(0xFFAB)
  })
  it('round-trip: read after write', () => {
    const val = writeSubBus(0, 0x42, 4, 8)
    expect(readSubBus(val, 4, 8)).toBe(0x42)
  })
})

describe('clampToWidth', () => {
  it('clamps 0x1FFFF to 16 bits', () => {
    expect(clampToWidth(0x1FFFF, 16)).toBe(0xFFFF)
  })
  it('no change when value fits', () => {
    expect(clampToWidth(0xFF, 16)).toBe(0xFF)
  })
})
```

## Pitfalls

- **Backward compatibility**: All existing wires have no `width` field. Treat `undefined` as `width: 1`. Existing tests must not break.
- **JavaScript bitwise ops**: `|`, `&`, `^`, `~`, `<<`, `>>` operate on 32-bit signed integers. Use `>>> 0` to coerce to unsigned where needed. Values above 32 bits are not needed for Project 1 (max 16-bit).
- **P05-03 dependency**: If topological eval is not merged yet, integrate with the existing `simulationTick`. The bus ops are pure functions — they work regardless of eval strategy.
- **`Wire.width` vs endpoint widths**: A wire's width should match the narrower of its two endpoints. Validation is a nice-to-have but not required for this ticket — the implementer should document the expected invariant.
- **Immer drafts**: When modifying `state.outputNodes[i].value = newValue` inside `set()`, Immer handles the mutation. Don't spread.
